===========================
==== Testers Documentation ====
===========================

tester.js contains 2 parts: Tester for static abilities and tester for dynamic abilities. Each part contains 10 tests.

Static tester - examins the following things:

1) Sending a legal request - Checks that the server deals with a legal http request.

2) Unexisting file Request - Checks that the server returns "404 Not Found" status when the request is for a file that doesn't exists in the directory.

3) Too many arguments in the request initial line - Checks that the server doesn't accept requests that contains more than the 3 arguments in the conventional http protocol.

4) Too little arguments in the request initial line - Checks that the server doesn't accept requests that contains less than the 3 arguments in the conventional http protocol.

5) Illegal HTTP version - Checks that the server doesn't accept requests from an illegal http version.

6) Illegal HTTP version parameter - Checks that the server doesn't support http requests that has an illegal parameter at the http version.

7) Not a GET method - Checks that the server doesn't support http requests that are not GET requests.

8) Illegal method parameter - Checks that the server doesn't support http requests that has an illegal method parameter.

9) Ignore multiple spaces - Checks that the server deals with multiple spaces, that those spaces don't harm the parsing of the request. A legal request will be responded correctly.

10) Send multiple requests at once - Checks that the server deals with input data that contains several http requests.

Dynamic tester - examins the following things:

1) Params of Request - Checks that the server succeed in getting the params from the URL.

2) query of Request - Checks that the server parse the query correctly from the URL.

3) Body of Request - Checks that the server parse the request body correctly.

4) Cookies of Request - Checks that the server parse the cookies correctly.

5) Path of Request - Checks that the server parse the path correctly.

6) Param function of Request - Checks that the server's param() function returns value of the given key parameter.

7) Is function of Request - Checks that the server's is() function returns whether the value of the Content-Type header is equal to the function parameter.

8) Set function of Response 2 parameters - Checks that the server's set() function sets the value of the given header parameter to be the second parameter.

9) Set function of Response JSON parameter - Checks that the server's set() function deals correctly with parameters that comes in a JSON object.

10) Status function of Response - Checks that the server's status() function sets the response status.
